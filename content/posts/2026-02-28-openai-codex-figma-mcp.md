## 3줄 요약
- OpenAI가 Codex와 Figma를 연결하는 **Figma MCP Server**를 공개했다.
- Figma 프레임 링크를 Codex에 전달하면 디자인 정보를 읽어 코드로 구현하는 흐름이 가능해졌다.
- 반대로 실행 중인 UI를 캡처해 Figma 파일로 변환하는 등, 디자인↔코드 왕복 루프가 빨라질 전망이다.

## 핵심 포인트
1. **디자인→코드 변환 가속**: Figma 프레임 기반으로 UI 구조와 스타일 정보를 코드 생성에 직접 활용.
2. **코드→디자인 역변환 지원**: 실행 화면 캡처를 Figma 에셋으로 연결해 협업 흐름 단축.
3. **기존 디자인 시스템 재활용**: 팀이 이미 쓰던 컴포넌트/토큰 체계를 살린 채 구현 가능.
4. **프로토타입-개발 간격 축소**: 시안 공유 후 구현까지의 커뮤니케이션 비용 절감 효과 기대.

## 코멘트
이 업데이트의 실무적 의미는 “디자인 핸드오프 문서” 중심 프로세스에서, **디자인 자산 자체를 개발 입력으로 쓰는 프로세스**로 이동한다는 점이다.

특히 제품팀 입장에서는 다음 효과가 크다.
- 디자이너: 시안 전달 이후 구현 일관성 검수 부담 감소
- 개발자: 스펙 해석보다 구현/품질 개선에 시간 집중
- 팀 전체: 반복적인 픽셀 맞춤 커뮤니케이션 감소

결국 경쟁력은 단순 생성 속도보다, **디자인 시스템과 코드베이스를 얼마나 안정적으로 왕복 동기화하느냐**에서 갈릴 가능성이 높다.
